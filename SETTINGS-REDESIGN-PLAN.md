# Settings 页重设计方案

## 现状诊断
1. 当前页面同时存在两层导航：外层竖向分类 Tab（`web/src/features/settings/index.tsx:88-131`）和内层“媒体源 / 媒体库”Tab（`web/src/features/settings/components/library-management-panel.tsx:239-279`），用户需要先选大类，再选子类，路径过长。
2. 页面把“真实后台配置”与“展示型偏好项”混在一起。`媒体库` 分区有真实的 query/mutation 和 CRUD 流程（`library-management-panel.tsx:57-228`），但 `播放与设备`、`通知与任务`、`账号与安全` 只是默认值控件，没有持久化状态或保存动作（`index.tsx:140-341`），视觉上却被赋予了同等权重。
3. 多个分区都使用“主卡片 + 右侧摘要卡”双栏结构（例如 `index.tsx:141-203`、`208-275`、`279-340`），摘要内容大多只是重复左侧已展示的信息，造成信息冗余。
4. `媒体源` / `媒体库` 目前以大卡片网格展示（`media-sources-tab.tsx:45-99`、`libraries-tab.tsx:49-115`），每张卡片都包含标题、说明、信息块、按钮，视觉密度过高；当条目变多时，扫描和管理效率会明显下降。
5. 页头同时放置“返回首页”和“元数据治理”按钮（`index.tsx:65-85`），再叠加左侧菜单和右侧内容，使顶部成为又一个决策区。

## 改版目标
1. 把 `/settings` 从“堆卡片的设置页”改成“管理员设置工作台”。
2. 优先突出真实可操作的资源管理：媒体源、媒体库、扫描入口。
3. 将次级工作区入口收敛成清晰的跳转模块，而不是并列塞进主设置流。
4. 降低重复摘要、边框、说明文案带来的视觉噪音。
5. 保证桌面端和移动端都能快速定位常用操作。

## 新的信息架构

### 1. 页面改成三段式结构
1. 顶部 Hero：标题、1 句页面定位、返回首页按钮。
2. 主工作区：`媒体资源管理`，作为页面唯一主内容。
3. 次级区域：`系统偏好` + `相关工作台入口`。

### 2. 调整分类优先级
1. `媒体资源管理` 保留为主区，承载媒体源与媒体库。
2. `计划任务` 与 `元数据治理` 从页头按钮/摘要入口改为“相关工作台”卡片区，与 `schedules`、`metadata` 页面当前工作台风格保持一致。
3. `播放与设备`、`通知与任务`、`账号与安全` 改成收敛式偏好区，不再和媒体库管理并列抢主视觉。

## 具体布局方案

### A. 顶部区域
1. 保留页面标题 `设置`，但把说明从“左侧切换设置分类...”改为更明确的工作台描述，例如“管理媒体来源、库挂载与系统偏好”。
2. 页头只保留一个主返回动作：`返回首页`。
3. `元数据治理`、`计划任务` 不再放在页头，移入下方“相关工作台”。

### B. 媒体资源管理主区
1. 取消外层左侧竖向 Tab，首屏直接展示 `媒体资源管理`。
2. 保留 `媒体源 / 媒体库` 二级切换，但从当前嵌套 Tab 改成更轻量的 segmented control，放在主卡片标题区。
3. 将 `媒体源`、`媒体库` 从卡片网格改成“列表行 + 右侧操作”布局：
   - 左侧显示名称、类型、来源、路径等核心信息。
   - 中间显示状态/标签。
   - 右侧放 `编辑 / 扫描 / 删除`。
4. 列表顶部新增一行概览指标：
   - 媒体源数量
   - 媒体库数量
   - 可扫描库数量 / 最近动作提示
5. 删除当前重复的右侧摘要卡，改成一个更小的“操作提示 / 空状态说明”模块，仅在需要时展示。

### C. 系统偏好区
1. 将 `播放与设备`、`通知与任务`、`账号与安全` 改为 3 个纵向 section 或 accordion，而不是主内容区的独立大页签。
2. 每个 section 采用“标题 + 简短说明 + 2~3 个控件”的轻表单结构，弱化大面积卡片和分隔线。
3. 因为当前这些控件没有真实持久化逻辑，所以视觉上应明确它们属于“偏好项/占位配置”，不要和真实 CRUD 区域同权。
4. `通知与任务` 中关于计划任务的说明块，改成简短的“前往任务工作台”入口，不再占据大块说明容器。

### D. 相关工作台入口
1. 在页面底部或右侧单独放 `相关工作台` 区块。
2. 仅保留两张入口卡：
   - `计划任务工作台`
   - `元数据治理工作台`
3. 每张卡片只包含标题、1 句说明、跳转按钮。

## 组件与文件调整建议
1. `web/src/features/settings/index.tsx`
   - 重写页面骨架。
   - 去掉当前外层竖向 `Tabs`。
   - 新增 `媒体资源管理`、`系统偏好`、`相关工作台` 三段式布局。
2. `web/src/features/settings/components/library-management-panel.tsx`
   - 保留数据获取与 CRUD 逻辑。
   - 重排成“概览条 + 分段切换 + 列表内容”。
3. `web/src/features/settings/components/media-sources-tab.tsx`
   - 从网格卡片改为列表行组件。
4. `web/src/features/settings/components/libraries-tab.tsx`
   - 从网格卡片改为列表行组件。
5. `web/src/features/settings/components/settings-aside-card.tsx`
   - 删除或改造成更轻量的 `SummaryStat` / `HintCard`，不再作为每个分区的固定右栏。
6. 可新增更小粒度展示组件：
   - `settings-section.tsx`
   - `settings-workspace-link-card.tsx`
   - `resource-list-item.tsx`
   - `settings-stat-strip.tsx`

## 视觉方向
1. 参考项目里已经较清晰的“工作台”页面风格，而不是继续堆叠半透明大卡片。
2. 减少同时出现的圆角层级、边框、阴影、分隔线数量。
3. 让主区最多只出现一个重容器，次级区使用更轻的 section/card。
4. 移动端优先纵向流：
   - 顶部 Hero 折叠为紧凑头部
   - `媒体源 / 媒体库` 分段切换横向滚动
   - 列表项改为上下信息堆叠，按钮底部排列

## 实施步骤
1. 先重构 `index.tsx` 页面骨架，移除外层纵向 Tab 和重复摘要区。
2. 再重构 `library-management-panel.tsx` 的头部信息结构与二级切换位置。
3. 将 `media-sources-tab.tsx`、`libraries-tab.tsx` 改成列表式呈现。
4. 把 `播放/通知/安全` 三块压缩成轻量偏好 section。
5. 最后补上“相关工作台”入口区，并统一页面间跳转文案。

## 验证方式
1. 在 `web/` 下运行 `pnpm typecheck`。
2. 运行 `pnpm build`，确认没有布局改动带来的编译问题。
3. 手动检查 `/settings`：
   - 桌面端首屏是否能直接看到媒体资源管理
   - 移动端切换和操作按钮是否不拥挤
   - 从设置页跳转 `schedules` / `metadata` 是否仍然顺畅

## 需要确认的取向
1. 我建议采用“设置工作台”方向：把 `媒体资源管理` 做成主区，把其他偏好项收敛成次级 section。
2. 如果你更想保留传统“分类设置页”模式，也可以改成更简洁的左侧导航版，但清爽度会不如前者。
