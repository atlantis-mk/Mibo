## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| provider source rows -> catalog normalization helpers | untrusted provider data becomes persisted catalog images/people/tags |
| provider refresh -> existing manual/local rows | automated replacement must not erase curated rows from other sources |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01 | T | `ReplaceItemImages` / `ReplaceItemPeople` / `ReplaceItemTags` | mitigate | scope deletions to `item_id + source_id` so provider reruns cannot tamper with manual or local rows from other sources |
| T-15-02 | D | normalized image selection | mitigate | elect exactly one selected image per `image_type` using deterministic ordering so repeated refreshes do not thrash canonical artwork |
| T-15-03 | I | normalized relationship tables | accept | raw provider payloads remain in `metadata_sources`; normalized tables store only canonical image URLs, names, roles, and tags needed for app behavior |
</threat_model>
