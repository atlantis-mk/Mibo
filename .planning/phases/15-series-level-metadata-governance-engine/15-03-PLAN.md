<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TMDB payloads -> metadata orchestration | untrusted provider data becomes canonical series/season/episode state |
| provider refresh -> locked/manual canonical fields | automation must not overwrite curator-owned fields |
| metadata orchestration -> catalog projections | partial writes must not leave rollups/search documents stale |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-07 | T | `RefreshCatalogSeriesMetadata` | mitigate | route canonical field changes through `catalog.ApplyField(... Force:false)` so locked/manual field states block provider overwrites automatically |
| T-15-08 | T | provider identity on catalog roots | mitigate | write the root match through `SetExternalID` with `provider="tmdb"` and `provider_type="series"` so one canonical external identity wins under the existing unique index |
| T-15-09 | D | projection freshness after refresh | mitigate | call `RefreshItemProjection` after the full root-series write finishes so rollups/search documents rebuild from completed canonical rows rather than partial state |
</threat_model>
