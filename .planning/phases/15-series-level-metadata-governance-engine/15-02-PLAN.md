## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| provider season/episode evidence -> catalog hierarchy helpers | provider data becomes durable season/episode rows |
| catalog hierarchy helpers -> existing scanned/backfilled rows | automated upserts must not duplicate or overwrite established slots incorrectly |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-04 | T | `UpsertGovernedEpisode` | mitigate | resolve existing rows by canonical `(parent_id, index_number)` slot before creating provider-only rows so provider refresh cannot duplicate local episode identity |
| T-15-05 | T | synthetic provider-only paths | mitigate | use deterministic `catalog://series/...` synthetic paths only for newly created provider-only rows; preserve existing scanned/backfilled paths when reusing a row |
| T-15-06 | D | `ReconcileEpisodeAvailability` | mitigate | compute availability from active asset/file links plus air date so missing provider episodes cannot be misreported as playable |
</threat_model>
