<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| storage events / scan jobs → reindex jobs | Untrusted path activity becomes queued projection work. |
| authenticated progress updates → search projection | User-scoped watch state crosses into shared discovery reads and must stay correctly keyed. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-03-01 | T | `library/scan_run.go` | mitigate | Queue projection rebuilds only after successful scan completion, with deduped job keys, so partial scan failures cannot stamp stale projection state as current. |
| T-08-03-02 | I | `progress/service.go` | mitigate | Reindex using the current user-scoped watch semantics after progress persistence so one user’s update cannot be interpreted through stale or global cached state. |
| T-08-03-03 | D | `worker/worker.go` | mitigate | Run bulk refresh through job kinds rather than inline request paths to avoid request-time denial of service during library scans or metadata bulk updates. |
</threat_model>
