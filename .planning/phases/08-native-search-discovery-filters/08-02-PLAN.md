## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TMDB response → metadata service | Third-party payload becomes trusted product metadata. |
| metadata service → search projection | Canonical rows are transformed into a read model used by user-facing discovery. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-02-01 | T | `metadata/service_match.go` | mitigate | Normalize TMDB country/rating fields before persistence and keep writes in the single `applyDetail` path so malformed payloads cannot create divergent discovery state. |
| T-08-02-02 | I | `search/service.go` | mitigate | Populate projection rows only from canonical `MediaItem` records, not directly from request parameters, so user-facing discovery cannot leak unvalidated transient values. |
| T-08-02-03 | D | `database/database.go` | accept | Auto-migrating one additional projection table is low-risk for this phase; no external network amplification or unbounded user input is introduced. |
</threat_model>
