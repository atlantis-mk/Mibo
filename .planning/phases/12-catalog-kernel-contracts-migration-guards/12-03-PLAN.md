<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| library scan workflows → jobs queue | Large or repeated refresh requests are persisted and later executed asynchronously |
| worker → projection tables | Background jobs rewrite rollup/search projection state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-07 | D | catalog projection jobs | mitigate | Use dedicated item/library-scoped payloads and unique enqueue helpers instead of unbounded generic refresh requests. |
| T-12-08 | T | `item_rollups` / `catalog_search_documents` | mitigate | Rebuild only the targeted item/library scope and cover the resulting state with worker integration tests. |
| T-12-09 | R | asynchronous projection updates | mitigate | Route all refreshes through named job kinds and persisted jobs so execution remains auditable in the existing worker model. |
</threat_model>
