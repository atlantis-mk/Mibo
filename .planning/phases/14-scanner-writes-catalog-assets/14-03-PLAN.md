<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| queued probe payload -> worker | worker must trust only typed inventory-file probe payloads |
| ffprobe output -> media_stream / media_asset updates | external process output becomes persisted technical metadata |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-07 | T | probe job payload | mitigate | use a dedicated `probe_inventory_file` payload with one `inventory_file_id` field, decode it in `worker.go`, and fail invalid JSON before any DB mutation |
| T-14-08 | I | ffprobe persistence | mitigate | persist only normalized stream fields and compact technical summary JSON; do not store raw command output blobs in catalog metadata sources |
| T-14-09 | D | scan/probe coupling | mitigate | keep probing asynchronous through jobs/worker so scan traversal still finishes quickly and large media libraries do not block request/worker loops |
</threat_model>
