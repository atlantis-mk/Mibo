| filename/path classification -> hierarchy creation | untrusted storage names determine series/season/episode placement |
| scan traversal -> persistence | every scanned object can create or reuse durable rows |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-14-04 | T | episode classification | mitigate | parse only explicit supported range forms and canonical season/episode keys in `scan_classify.go`; do not derive hierarchy from arbitrary trailing tokens |
| T-14-05 | D | scan write amplification | mitigate | keep one asset per scanned file and reuse canonical item rows by hierarchy path so duplicate rescans do not explode item counts |
| T-14-06 | R | cutover auditability | mitigate | add integration tests that assert legacy `media_items` / `media_files` counts stay zero for newly scanned fixtures |
</threat_model>
