<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| storage object metadata -> scan writer | scanner consumes provider-controlled file names, paths, and identity evidence |
| local evidence payload -> catalog metadata_sources | persisted evidence must stay compact and non-sensitive |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-01 | T | scanner hierarchy writer | mitigate | normalize library-scoped item paths and generate canonical season/episode keys in `scan_catalog.go` instead of trusting raw file paths for hierarchy rows |
| T-14-02 | I | scanner local evidence payload | mitigate | persist only allowlisted local evidence keys in `metadata_sources.payload_json`; do not store arbitrary provider blobs or ffprobe output in this source record |
| T-14-03 | D | library service wiring | mitigate | reuse shared `catalog` and `inventory` services from the existing DB handle instead of creating ad hoc per-file transactions that could fragment scan throughput |
</threat_model>
