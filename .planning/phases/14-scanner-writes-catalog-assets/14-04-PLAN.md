<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| provider file listing -> missing-file cleanup | an incomplete or delayed listing can look like content deletion |
| stable identity evidence -> file reuse | incorrect identity reuse could bind the wrong asset to an item |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-10 | T | missing-file cleanup | mitigate | change cleanup to status/availability flips only and never delete `catalog_items`, `asset_items`, or `metadata_sources` when a file disappears |
| T-14-11 | S | stable-identity reuse | mitigate | reuse existing file/asset rows only when the same `stable_identity_key` matches; path-only differences without identity must not silently merge unrelated assets |
| T-14-12 | D | version availability rollup | mitigate | recompute leaf availability from remaining available assets so one missing version cannot incorrectly mark a still-playable episode unavailable |
</threat_model>
