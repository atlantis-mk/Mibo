## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin browser → schedule APIs | Untrusted schedule definitions and actions cross the network boundary. |
| schedule service → jobs queue | API-triggered runs must enqueue validated job payloads, not arbitrary client input. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-04-01 | S | `handlers_schedules.go` | mitigate | Require `requireUser` on every schedule route before CRUD/toggle/run-now/history access. |
| T-10-04-02 | T | schedule mutation endpoints | mitigate | Use strict `decodeJSON` plus explicit schedule-service validation for kind/scope/frequency combinations before enqueueing work. |
| T-10-04-03 | D | run-now endpoint | mitigate | Return accepted async job metadata immediately and never execute maintenance inline inside the request path. |
</threat_model>
