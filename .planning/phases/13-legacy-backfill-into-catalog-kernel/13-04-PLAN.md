<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| weak legacy TV identity -> canonical series grouping | ambiguous legacy episode metadata crosses into permanent hierarchy writes |
| duplicate logical episode rows -> canonical episode item | multiple legacy rows may try to claim one season/episode slot |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-10 | T | series grouping logic | mitigate | prefer provider-backed TV series IDs, fall back to normalized `SeriesTitle` only when provider identity is absent, and emit `conflict` entries for missing/ambiguous identity |
| T-13-11 | R | duplicate episode resolution | mitigate | persist `duplicate_episode_candidate` entries whenever multiple legacy rows claim one canonical season/episode slot so operators can inspect every merge decision |
| T-13-12 | I | orphan-file reporting | accept | orphan file report entries expose only library/file identifiers and paths, which are already operator-visible inside the system and needed for repair |
</threat_model>
