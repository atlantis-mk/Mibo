| provider identity strings -> catalog external IDs | overloaded legacy metadata must be normalized before persistence |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-13-07 | T | movie identity reuse | mitigate | query by legacy library + `source_path` before creating a new movie row and reuse existing inventory/link rows through service upserts |
| T-13-08 | I | migrated metadata evidence | mitigate | persist only compact legacy provenance JSON in `metadata_sources`; do not dump arbitrary legacy blobs or unrelated secrets into evidence payloads |
| T-13-09 | R | per-row migration audit | mitigate | append a `success` or `skipped` report entry for every processed movie row so reruns remain explainable |
