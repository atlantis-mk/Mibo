<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authenticated admin -> backfill HTTP endpoints | privileged operation starts durable migration work |
| queued job payload -> worker execution | worker must trust only typed payloads and existing run IDs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-04 | S | backfill trigger API | mitigate | require `requireUser` before request decode and derive `triggered_by_user_id` from the authenticated session instead of allowing the client to supply it |
| T-13-05 | T | queued backfill job payload | mitigate | enqueue only `catalog.LegacyBackfillPayload` with a run ID created by the catalog service; worker decodes the typed payload and fails invalid JSON early |
| T-13-06 | D | HTTP trigger path | mitigate | return `202 Accepted` after enqueueing so long-running backfill work executes in the worker, not inside the request lifecycle |
</threat_model>
