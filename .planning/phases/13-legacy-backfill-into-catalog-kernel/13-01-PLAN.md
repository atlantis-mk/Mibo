<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator request -> migration run store | authenticated users trigger privileged migration work |
| legacy DB rows -> report persistence | malformed legacy data crosses into durable audit records |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | S | migration run creation helpers | mitigate | keep run creation inside catalog service methods that require a recorded `triggered_by_user_id`; later HTTP plan must source it from `requireUser` rather than client JSON |
| T-13-02 | T | report aggregation | mitigate | compute aggregate counters from persisted `CatalogMigrationEntry` rows in `finalizeLegacyBackfillRun` instead of trusting request payloads or worker memory |
| T-13-03 | I | report entry payloads | mitigate | limit persisted report columns to IDs, storage path, title, message, and `details_json`; do not copy raw provider payloads or secrets into report rows |
</threat_model>
