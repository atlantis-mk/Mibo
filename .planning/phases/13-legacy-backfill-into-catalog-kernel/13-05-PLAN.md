<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| legacy playback progress -> user_item_data | user-state migration crosses from legacy IDs to catalog IDs |
| successful worker run -> migration settings | privileged runtime updates durable cutover state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-13 | T | progress mapping | mitigate | resolve `asset_id` only through the migrated catalog/item/file links built by earlier slices and upsert `UserItemData` on its unique key to avoid duplicate user-state rows |
| T-13-14 | E | migration state update | mitigate | in `worker.go`, update only `catalog_backfill_completed_at` after a successful run and preserve the previously stored `catalog_read_enabled` and `legacy_cleanup_completed_at` values |
| T-13-15 | D | stale projections after migration | mitigate | call `RefreshLibraryProjection` for each touched library before finalizing a successful run so downstream read phases do not consume stale rollups/search docs |
</threat_model>
