## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| storage event input → listener service | Untrusted event kind/path data crosses into durable refresh intent. |
| listener service → jobs table | Bad merge logic can flood or widen background work beyond one library. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01-01 | D | `internal/listener/service.go` merge window | mitigate | Persist a concrete 15-second debounce window and merge only by `library_id + normalized_root`, with tests that prevent duplicate event storms from enqueuing unlimited work. |
| T-11-01-02 | T | ancestor-promotion logic | mitigate | Promote only to the minimal safe common ancestor inside the library root and mark unsafe cases as `fallback_full_sync=true` instead of widening beyond the boundary. |
| T-11-01-03 | E | listener policy placement | mitigate | Keep all event-policy code inside `mibo-media-server/internal/listener`; do not delegate trust decisions to OpenList or any external bus per D-13 and D-14. |
</threat_model>
