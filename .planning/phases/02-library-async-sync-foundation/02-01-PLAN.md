<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin client → HTTP API | Untrusted request payloads cross into authenticated settings and jobs endpoints |
| worker loop → database/jobs queue | Background scheduler can enqueue large amounts of work if interval logic is wrong |
| HTTP API → settings persistence | Persisted refresh cadence affects future autonomous execution |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | T | `router.go` scan settings handlers | mitigate | Require authenticated user access, use strict JSON decode, and reject `refresh_interval_hours` outside `1..720` before persistence |
| T-02-02 | D | `worker.go` scheduled scan trigger | mitigate | Read one interval value, skip disabled values, and enqueue through `QueueLibraryScan`/`EnqueueUnique` so scheduled refresh cannot flood duplicate jobs |
| T-02-03 | I | `jobs/service.go` filtered jobs listing | accept | Job metadata is admin-only operational data; keep endpoint behind existing auth and return only stored job fields already exposed elsewhere |
</threat_model>
