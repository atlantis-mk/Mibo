<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| governance mutation inputs -> catalog tables | user-controlled lock/image/asset requests become durable catalog changes |
| provider evidence rows -> governance workspace DTO | internal evidence crosses into admin-facing governance reads |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-03 | T | `UpdateFieldLock` | mitigate | reuse `ApplyField(... Force:true)` with the current canonical value so lock toggles do not tamper with the field value itself |
| T-16-04 | T | `SelectImage` / `LinkAsset` | mitigate | scope updates and deletes by explicit `item_id + image_type` or `asset_id + item_id` keys so one governance action cannot alter unrelated candidate rows or asset links |
</threat_model>
