<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser governance hooks -> authenticated catalog APIs | untrusted route params and UI actions become item/asset mutation requests |
| provider evidence payload -> normalized frontend model | provider-derived summaries cross from backend JSON into rendered UI state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-01 | T | API helper layer | mitigate | centralize all governance routes in `web/src/lib/mibo-api.ts` with typed helper signatures so later panels cannot spoof arbitrary request shapes |
| T-19-02 | I | raw governance payloads | mitigate | normalize provider summaries into plain JSON/text models inside feature hooks and avoid passing raw `any` blobs into components |
| T-19-03 | E | item/asset mutation helpers | mitigate | require `itemId` and `assetId` to come from route/query state and keep helper names catalog-native so legacy id confusion cannot widen mutations |
| T-19-04 | D | query invalidation | mitigate | use stable governance-specific query keys so repeated mutations refetch only the intended workspace/detail views instead of triggering broad cache churn |
</threat_model>
