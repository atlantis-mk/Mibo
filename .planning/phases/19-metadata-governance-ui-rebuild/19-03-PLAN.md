<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| field edit controls -> mutation endpoint | user edits and lock toggles become server-side governance mutations |
| provider evidence summaries -> rendered UI | untrusted provider-derived content crosses into visible panels |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-08 | T | field-state edits | mitigate | only expose field keys returned by the workspace hook and route all writes through `updateCatalogFieldState` so components cannot invent arbitrary keys |
| T-19-09 | I | source-evidence panel | mitigate | render evidence payloads as plain text and collapsed `<pre>` JSON only, with no HTML injection path |
| T-19-10 | R | lock/edit metadata | mitigate | display `edited_by_user_id`, `edited_at`, `is_locked`, and `lock_reason` directly in the field-state panel so manual overrides remain attributable |
</threat_model>
