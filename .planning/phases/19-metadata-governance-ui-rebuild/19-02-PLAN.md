<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| route params and workspace filters -> item list/detail navigation | untrusted browser state controls which catalog item is opened |
| workspace cards -> user decision making | bounded governance summaries influence which item an admin edits |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-05 | T | workspace filters | mitigate | keep filters typed and item-scoped inside `useGovernanceWorkspace` so arbitrary URL/query injection does not widen fetch parameters silently |
| T-19-06 | E | route/id navigation | mitigate | parse `$id` with `Number(id)` and guard against non-positive ids before rendering detail flows so navigation cannot accidentally target invalid mutation contexts |
| T-19-07 | I | workspace cards | accept | workspace cards intentionally expose only governance-safe summary fields (status, image, identities, counts); no raw evidence payloads are shown here |
</threat_model>
