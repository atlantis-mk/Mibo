<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authenticated HTTP input -> audit/repair queue | operators can request integrity work that may mutate catalog rows |
| queued job payload -> worker execution | malformed or widened payloads could repair the wrong scope |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-03 | E | consistency operation endpoints | mitigate | require `requireUser` before queue/list/detail/repair handlers and return typed payloads instead of exposing raw jobs internals |
| T-20-04 | T | worker repair payloads | mitigate | validate run id, library scope, root path, and repair mode before dispatch so the worker cannot execute an unintended full-db mutation |
</threat_model>
