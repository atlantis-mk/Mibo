<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| catalog item request -> asset selection | client input chooses which item and optional asset should be considered for playback |
| storage provider -> playback decision | untrusted provider/file existence data determines whether playback stays direct, falls back, or becomes unplayable |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-01 | E | asset selection | mitigate | validate explicit `asset_id` against `asset_items.item_id = requested item id` before using it so callers cannot escalate into another item's file |
| T-17-02 | T | candidate ranking | mitigate | rank only `media_assets.status="available"` plus `asset_files.role="source"` rows and keep deterministic tie-breaks to avoid unstable or spoofable selection |
| T-17-03 | I | playback response payload | mitigate | expose only catalog/inventory identifiers and playback metadata; do not leak raw storage-provider payloads or unrelated file-system details |
| T-17-04 | D | storage-backed resolution | mitigate | collapse missing/unreadable source files into structured `unplayable` decisions so callers do not trigger repeated crashing retries on transient storage failures |
</threat_model>
