<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP path/query params -> playback handlers | untrusted client input chooses catalog item and optional asset ids |
| playback service -> returned URLs | handlers must expose only validated playback URLs with auth propagation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-05 | E | `/api/v1/items/{id}/playback` | mitigate | require authenticated user and forward `asset_id` only through the playback service's item-bound validation rather than trusting raw query params |
| T-17-06 | T | handler routing | mitigate | register only explicit item and asset playback routes and remove any need for legacy `file_id` query parsing in the new handlers |
| T-17-07 | I | response URL shaping | mitigate | continue to run returned paths through `buildPlaybackURL(...)` so access tokens are attached only to validated playback URLs, not arbitrary client-supplied paths |
</threat_model>
